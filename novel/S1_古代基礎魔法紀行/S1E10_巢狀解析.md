# S1E10｜〈巢狀解析〉—— JSON Nested Feature Extraction（2001）

## 一、港口海關

清晨，港口海關塔下排了十幾隊商隊。稅務官焦頭爛額——桌上攤開一份厚厚的卷軸。

稅務官見芙莉蓮一行，立刻招手：「芙莉蓮大人——這批貨單，十萬筆。上頭要我三日內算出每家商隊的風險係數——可這東西……」

他把卷軸翻開：最外一筆，裡頭嵌著三筆；三筆每筆又嵌著九筆；九筆再往下嵌更細的欄位。

修塔爾克看一眼就閉眼：「這、這是層中有層？」

費倫翻得很快：「訂單下分項目，項目下分包裹，包裹下是重量、來源地、申報日。」

稅務官點頭：「對——但我只會做單層的表。扁平化之後，很多東西都沒了。」

芙莉蓮半靠在海關塔的柱子上：「……費倫，妳來。」

費倫鞠躬。

---

## 二、一鍵扁平

費倫先試老法子。她閉目凝神，把整份貨單**一鍵扁平**——所有欄位展成一張巨大的單層表。

表浮在空中，邊角卻越來越淡——底層的子欄位像霧一樣散開，根本沒進到表裡。

稅務官看著那片空白，臉都白了：「十萬筆裡——至少八萬筆的子欄位不見了！」

費倫皺眉：「一鍵扁平化——把結構壓沒了。」

修塔爾克小聲：「沒、沒辦法展嗎？」

費倫搖頭：「不是展不開。是『直接扁平』把巢狀層級當成重複鍵，後來的覆蓋前面的。」

芙莉蓮淡淡：「……妳試了最懶的。」

費倫老實：「……是。」

「……懶有懶的代價。」芙莉蓮說，「八萬筆代價。」

---

## 三、遞迴的樹

費倫重新閉眼。這次她不再把整份貨單攤平，而是想像自己是一隻從最外層開始、一層一層往裡走的小蟲——每進一層，就把那一層的欄位名展成新的欄。

「……一層、一層——」她心中念著，「到最深處，再回來。」

空中浮出一棵樹——頂端是訂單，往下分枝是項目、包裹，最外的葉子是重量、來源地、申報日。每一個葉子都亮起一個獨立的欄。

但她還沒說真名。芙莉蓮開口：「……還差一步。」

費倫：「時間？」

「……嗯。只有結構不夠。」芙莉蓮說，「同一家商隊過去一個月、三個月的**趨勢**要留下來。」

費倫立刻在葉子上再加一層——每個葉子接上一條時間軸，依「過去 7 天平均」「30 天最大」「90 天變異」聚合起來。

整棵樹同時亮起。費倫睜眼：

「……JSON Nested Feature Extraction。」

十萬筆貨單一瞬間全部展開——結構完整、時間特徵齊備。稅務官倒抽一口氣。

---

## 四、不是扁平，也不是字串

費倫轉身：「師父——有兩條路我剛才沒走。」

「……嗯。」

「第一條：JSON 直接扁平化轉 CSV——**壓掉結構**。」

「……對。」

「第二條：整段 JSON 當字串，丟進 RNN——讓模型自己讀括號。」

芙莉蓮輕輕搖頭：「……更糟。」

「為什麼？」

「……RNN 不懂括號。」芙莉蓮說，「把結構化的東西當自然語言看——它只會背字元、不會看層級。」

費倫筆記：

> **JSON 特徵萃取 — 正解與歧路**
> ✅ 正解：**遞迴展開巢狀欄位 + 時間窗口聚合**。
> ❌ 歧路一：扁平化轉 CSV → 子欄位丟失、結構壓沒。
> ❌ 歧路二：整段 JSON 字串丟 RNN → 字元層級讀取、看不到結構。

---

## 五、書吏的亂碼

海關角落一位年輕書吏不服氣：「費倫小姐——我早試過『字串法』了。我把每筆 JSON 整段接成一條長字串，塞進 RNN——」

他手一揮，自家堂口的咒圈亮起。十萬條 JSON 字串像麵條一樣湧入——咒圈開始抖，浮出的特徵全是花花綠綠的亂碼。

書吏臉色發綠：「為什麼——」

「……字元層級的 RNN 只看得到『{』和『}』的前後。」芙莉蓮頭也不抬，「一筆一千字元，RNN 背到後半就忘了前半——結構全散。」

書吏再試一次，這次把括號去掉、只留鍵跟值——

費倫立刻插話：「那更糟。你把『哪個值屬於哪個鍵』的資訊也抹掉了。」

咒圈再抖兩下，徹底熄滅。

費倫筆記：

> 心像偏差一：整段 JSON 當字串 → RNN 讀字元、不讀層級 → 亂碼。
> 心像偏差二：把括號去掉再塞 RNN → 鍵值對應消失 → 更糟。
> 正解：**先用遞迴函式解析結構，再對數值葉節點做時間聚合**。

---

## 六、窗口的分寸

稅務官興奮地想複製費倫的方法：「我全用 7 天的平均就好了！」

芙莉蓮瞥他一眼：「……7 天？」

稅務官愣住：「不對嗎？」

「……看你要抓什麼。」芙莉蓮說，「查短線囤貨——7 天。查年度走私模式——90 天。查突發事件——1 天。」

費倫補：「**聚合單位要依情境調**。」

「依情境？」

「……一個特徵一個窗口。」芙莉蓮說，「同一組貨單，可以同時有 1 天、7 天、30 天、90 天四個版本的平均。」

稅務官瞪大眼：「同一個數字——要做四次？」

「……對。每個窗口都是獨立特徵。」芙莉蓮淡淡說，「讓模型自己選哪個對風險係數最有用。」

費倫筆記：

> 時間窗口聚合——**多尺度並存**：
> - 短窗（1–7 天）：捕捉突發變化。
> - 中窗（30 天）：捕捉月度節律。
> - 長窗（90 天）：捕捉季節趨勢。
> - 同一原始欄位可產生多個視窗版本，交給模型挑選。

---

## 七、博物館的一眼

午後，稅務官把風險係數算完，捧著冒煙的羊皮卷感激不已。一行人從港口走出，轉進舊城區，遠處博物館的圓頂在陽光下發白。

館長在門口攔住他們：「芙莉蓮大人——求您替本館設一道『一眼辨認』的護符。遊客一走進來，看到牆上的畫——護符就該告訴他這是什麼。」

費倫眼睛一亮：「影像分類？」

「……嗯。」芙莉蓮點頭。

修塔爾克好奇：「就、就看一眼就能說出是什麼？」

費倫微笑：「單一類別標籤。一張畫對應一個名字。」

館長又補：「能不能指出『畫面右下那隻怪獸』？」

費倫一愣，看向芙莉蓮。

芙莉蓮慢慢搖頭：「……下一章。」

費倫立刻明白——分類只回答「是什麼」，不回答「在哪裡」。

博物館的銅鐘響起。

## 芙莉蓮的冒險筆記（S1E10）

| 項目 | 內容 |
|---|---|
| 英文全名 | **JSON Nested Feature Extraction**（JSON 巢狀欄位特徵萃取） |
| 簡寫 | — |
| 中文意譯暱稱 | 巢狀解析 |
| 年代 | JSON 2001（Douglas Crockford 推廣） |
| 所屬科目 | 科目一｜特徵工程 |

**一句話口訣**
- **遞迴展開巢狀欄位 + 時間窗口聚合**——不是扁平化轉 CSV、不是把 JSON 字串丟進 RNN。

**考試重點速記**
- 正解：**遞迴函式展開每一層欄位**，讓每個葉節點成為獨立特徵；再對數值葉節點做**時間窗口聚合**（7 天平均、30 天最大、90 天變異等）。
- **歧路一**（易錯）：直接扁平化（flatten）轉 CSV → 重複鍵互相覆蓋、子欄位丟失。
- **歧路二**（易錯）：整段 JSON 字串餵 RNN → 字元層級讀取、看不到結構層級。
- **歧路三**（更糟）：去掉括號再餵 RNN → 鍵值對應消失。
- 聚合窗口**多尺度並存**：短、中、長窗同時產生獨立特徵，交模型挑選。
- 屬**特徵工程**類；常見題型：「面對層層巢狀的半結構化資料，下列何者為最佳處理方法」。

**心像崩潰對照**
- 一鍵扁平化 → 十萬筆中八萬筆子欄位消失（費倫示範）。
- 整段 JSON 字串塞 RNN → 咒圈讀字元、浮出亂碼（書吏示範）。
- 去掉括號再塞 RNN → 鍵值對應全失、更糟（書吏示範）。
- 正解：遞迴展開為樹 + 對葉節點做多尺度時間聚合。
