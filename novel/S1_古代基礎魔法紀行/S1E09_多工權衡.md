# S1E09｜〈多工權衡〉—— Multi-task Learning Loss Weight（1997）

## 一、雙燈試場

魔法學院的測驗場在市集盡頭——一座兩燈併立的石臺，紅燈寫「**影像**」，藍燈寫「**文字**」。兩盞要同時亮，才算過關。

主考官戴著單片鏡，對費倫點頭：「費倫小姐，這是『雙工令』。**同一道心像**要同時點亮兩盞燈。」

費倫鞠躬：「已經練了兩週。」

修塔爾克在旁小聲：「紅、紅燈平時都亮得很漂亮。」

費倫臉微紅：「藍燈……最近有點黯。」

芙莉蓮斜眼：「……多久了？」

「從第三天起。」費倫老實說，「紅燈越亮，藍燈越黯。」

主考官笑了一下：「那正是本試之考點。」

---

## 二、影像升，文字降

費倫先示範一次。她靜默凝神，雙手合十——影像先浮起，物件清晰分明；文字接著浮起，描述卻斷斷續續。

紅燈應聲大亮；藍燈搖曳兩下，幾乎熄滅。

費倫眉頭一皺：「師父——我確實把兩個任務都放在心裡。」

「……放了。」

「那為什麼一升一降？」

修塔爾克搔頭：「這……不會是學偏了吧？」

費倫追問：「會不會是**文字資料太少**？」

芙莉蓮吐出四個字：「……先查權重。」

費倫一愣：「權重？」

「……一升一降——**最可能是損失沒做權重平衡**。」芙莉蓮淡淡說，「不是資料少，不是模型小。」

費倫臉色一凜。

---

## 三、量級的秤

芙莉蓮指尖一劃，空中浮現兩個小秤——左秤堆著影像任務的損失數值，右秤放著文字任務的損失。

左秤沉得深；右秤幾乎翹到頂。

「……量級。」芙莉蓮說，「影像損失動輒幾十，文字損失常常零點幾。你加成一條——誰佔主導？」

費倫瞪大眼：「……大的那一條。」

「……對。」

「所以我這兩週——**模型一直在優化影像**，文字只是陪跑？」

「……嗯。」

費倫深吸一口氣，閉上眼。她把心像裡那條「把兩個損失直接加起來」的路抹掉，重新搭——

這一次，她在兩個損失前各放一個可調的係數：影像壓小、文字拉大，直到空中秤真正平衡。

紅燈依然亮；藍燈——穩穩浮起，和紅燈並肩。

費倫睜眼：

「……Multi-task Learning Loss Weight。」

兩盞燈同時大亮。

---

## 四、先查權重

主考官頷首：「費倫小姐，過。」

費倫轉身向芙莉蓮鞠躬：「師父——為什麼第一個問題就該是『權重』？」

「……一升一降有四個常見嫌疑。」芙莉蓮說，「資料不夠、模型容量不夠、學習率不對、**權重沒平衡**。」

「哪一個最先查？」

「……權重。」

「為什麼？」

「……最便宜、最常發生、最容易驗證。」芙莉蓮慢慢說，「資料要重標，模型要重訓，學習率要重掃——權重，**改一個係數就看結果**。」

費倫筆記：

> **一升一降 → 先查權重**
> 最可能原因：**損失函數未做權重平衡**。
> 再依序查：學習率、模型容量、資料量。

---

## 五、隔壁的 1:1

另一名院生忍不住上前：「費倫小姐——我練三週，紅燈比妳更亮、藍燈幾乎沒動。我就是把兩個損失直接加起來啊！」

費倫禮貌問：「你兩邊損失的量級——差多少？」

「沒算過。」院生聳肩，「反正 1:1 加，**公平**。」

芙莉蓮低聲：「……『公平』不是『平衡』。」

院生愣住：「差別？」

「……公平是數字一樣。」芙莉蓮接下去，「平衡是**影響力一樣**。你兩邊損失量級差十倍——1:1 加起來，小的那邊等於沒加。」

院生臉色一白，還是不服：「那我重練——」

他強行再發一次。紅燈暴亮到刺眼；藍燈在強光下徹底蓋過去——根本看不見。

主考官搖頭：「這就是『大任務吃掉小任務』。咒紋歪了。」

費倫筆記：

> 心像偏差：**1:1 直接加 → 損失量級差 → 大任務主導梯度 → 小任務被吞**。
> 咒紋表現：紅燈暴亮、藍燈消失 → 單任務過擬合、另一任務未訓練。

---

## 六、調的分寸

修塔爾克小聲：「師父——那反過來把文字壓大，影像就會變弱？」

「……會。」芙莉蓮點頭，「調過頭——影像也失準。權重是**秤**，不是蹺蹺板。」

費倫補一句：「要讓兩邊同時可用，不是此消彼長。」

「……嗯。」

修塔爾克又問：「那權重怎麼定？」

「……最簡單——**以量級倒數**當初值。」芙莉蓮說，「想再細——看兩邊的梯度大小動態調。」

費倫筆記：

> 實務起手：
> - 粗估：`w_i ≈ 1 / loss_i 的量級`，讓兩邊影響力相近。
> - 細調：依梯度範數或驗證集分數動態調整。
> - **禁忌**：1:1 直接相加、不問量級。

---

## 七、層層貨單

走出學院，海風又鹹起來。領隊等在門口，這回輪他滿臉為難：「芙莉蓮大人——港口海關要我們交貨單，格式是**層層巢狀**，一筆裡套三筆、三筆裡再套九筆……」

修塔爾克接過一看，眉頭皺成山：「這、這是什麼寫法？」

「叫 JSON。」領隊苦笑，「一層扣一層，展不開。」

費倫眼睛一亮：「是巢狀欄位——要遞迴展開。」

芙莉蓮嚼著剩下的糖：「……明天的海關，費倫，你來。」

「嗯。」

「……不要一鍵扁平化。」芙莉蓮補一句，「也不要把 JSON 字串整條塞進 RNN。」

修塔爾克一頭霧水：「那怎麼辦？」

芙莉蓮不回。只把糖紙一揉，往海的方向丟。

夕陽把紙團拋成一道小小的弧。

## 芙莉蓮的冒險筆記（S1E09）

| 項目 | 內容 |
|---|---|
| 英文全名 | **Multi-task Learning Loss Weight**（多任務學習損失權重） |
| 簡寫 | MTL Loss Weight |
| 中文意譯暱稱 | 多工權衡 |
| 年代 | 1997（Caruana 綜整 MTL） |
| 所屬科目 | 科目一｜MLOps／多任務學習 |

**一句話口訣**
- **多任務一升一降 → 最可能原因：損失函數未做權重平衡**。

**考試重點速記**
- 現象：一個任務分數上升、另一個下降 → **不是資料少、不是模型小；先查權重**。
- 原因：兩個損失**量級不同**，直接相加 → 大任務主導梯度，小任務被壓。
- 初步解法：以**量級倒數**為初值（`w_i ≈ 1 / loss_i 量級`）；進階以梯度範數／驗證分數動態調。
- **公平（1:1）≠ 平衡**：平衡指「對梯度的影響力一樣」，非數字一樣。
- 診斷排序（最便宜先）：**權重 → 學習率 → 模型容量 → 資料量**。

**心像崩潰對照**
- 兩個損失 1:1 相加 → 大量級吃小量級 → 紅燈暴亮、藍燈不見（院生示範）。
- 權重調過頭 → 另一邊反失準 → 權重是秤、非蹺蹺板。
- 先怪資料量 → 忽略最便宜的權重檢查 → 走遠路（費倫初始示範）。

**下章預告**：**JSON Nested Feature Extraction**（2001）——港口海關的巢狀貨單。不要一鍵扁平化，也不要整串 JSON 塞進 RNN。
