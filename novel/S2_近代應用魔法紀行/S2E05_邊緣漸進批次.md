# S2E05｜〈邊緣／漸進／批次〉—— Edge Deployment（2016）・Phased Rollout・Batch Size & GPU Memory

---

　　遠境山脈深處，一座貼著岩壁搭建的木造醫療站。屋頂漏著風，屋內的燈油忽明忽暗，外頭山霧重到連門口的招旗都看不清。

　　「這……這一路爬得真夠嗆。」修塔爾克揹下大背包，喘了兩口氣。

　　「辛苦您了，修塔爾克先生。」費倫遞過水囊。

　　芙莉蓮已經先一步坐在診療台上，翹著腿翻卷宗。

　　醫療站的老醫師從屋後端出熱湯，雙手纏著兩層繃帶。

　　「三天前外頭就斷了通訊。」老醫師搖頭，「山路塌，鴿哨也收不到。」

　　「所以雲端模型——連不上？」費倫皺眉。

　　「連不上。病人卻一個比一個重。」

---

　　費倫把隨身的水晶片平放在診療台上。

　　「芙莉蓮大人，我把南境醫療塔訓好的那隻主模型搬過來。」

　　「搬得動？」

　　「……它三百層。」

　　「你這台——只有指甲蓋大。」

　　水晶片在費倫手中微微發熱。她閉眼嘗試——整座主模型的輪廓才浮到一半，水晶片邊緣就焦黃冒煙。

　　顯存爆了。

　　「……唔。」費倫停手，「超出水晶片容量了。」

　　「要不要減一點樣本？」老醫師擔心地問。

　　「不。」芙莉蓮搖頭，「減資料——就等於少看病人。」

　　費倫低頭思索。問題不在一處——三處：設備小、斷雲、批次太大。

---

　　修塔爾克把戰斧放下，將地上那個大背包攤開——這是他在前線練出來的工兵本能。

　　背包從外到內一層層：外袋繩索火石、中袋止血布、內袋三天份乾糧。

　　「你們看。」他指著外袋，「戰士打仗時——最常用的東西，裝最外面。不然來不及拿。」

　　「意思是……」費倫抬眼。

　　「模型要跟病人放一起。不要放在雲端。」

　　芙莉蓮淡淡地接話。

　　「……Edge Deployment。」

　　修塔爾克愣了半秒，才反應過來那正是他剛才講的。

　　「對、對啦。離資料源最近——手機、機器、裝置上直接跑。」

　　「降延遲，少吃頻寬。」費倫順著寫進筆記。

　　「……還不怕斷線。」芙莉蓮補一句。

---

　　下一個難題——病人來自四科：骨傷、產科、腸胃、急診。

　　費倫原本打算一次就把四科的診斷模型全部接上線。

　　「不行。」修塔爾克立刻搖頭。

　　「為什麼？」費倫疑惑。

　　「新兵一次拉上前線——會全死。」

　　他把戰斧往地上一頓，沙地上畫了一條線。

　　「先從一個科開始。譬如……先骨傷。死亡風險最低，例子最多。」

　　「然後呢？」

　　「穩了，再擴到腸胃、產科。」修塔爾克頓了頓，「急診——放最後。」

　　「Phased Rollout。」芙莉蓮接道，「漸進式部署。醫療從單一專科起步——從急診開始是錯的，全院同步也是錯的。」

　　老醫師連連點頭。

　　「去年隔壁山谷就栽過這個跟頭——新設備一夜鋪滿全院——急診先爆，其他科跟著連鎖崩。」

　　「所以要漸進。」修塔爾克握緊斧柄，「戰術也是這樣。」

---

　　最後一個問題——費倫還是爆顯存。

　　她咬著下唇，盯著焦黃的水晶片發愁。

　　「要不要再少看一點病人？」老醫師試探地問。

　　「不減資料。」芙莉蓮合上卷宗，「——減批次。」

　　「批次？」費倫回頭。

　　「一次送進模型多少樣本——那個。」

　　修塔爾克又從背包裡掏出三個更小的布袋。

　　「像這樣。」他把大背包裡的三天份乾糧拆成三個小袋，「同樣的量，分成幾趟揹——肩膀不崩。」

　　「Data Sharding。」費倫點頭，眼睛亮起來，「把資料分片分批餵。」

　　「對。」芙莉蓮輕「嗯」一聲，「GPU 不夠——減批次、加分片。不是減資料量。」

　　費倫重新把心像送進水晶片。這次她把 Batch Size 調小、樣本切成三分片逐批載入——水晶片穩穩地亮起來，不再冒煙。

　　老醫師長長地鬆了口氣。

---

　　修塔爾克合起背包，聽到身後費倫小聲的道謝。

　　「修塔爾克先生，謝謝您……今天講得比我想像中清楚。」

　　「……啊？」他耳根又紅了一圈，「我、我就是按戰士的習慣講啦。」

　　「戰士的習慣跟工程的道理——一模一樣啊。」

　　「……是、是嗎？」

　　芙莉蓮看著這兩人，面無表情翻了下一頁卷宗，不接話。

---

　　夜裡，醫療站的油燈穩了下來。

　　費倫翻著筆記，忽然開口。

　　「芙莉蓮大人，下一站呢？」

　　「……畫坊。」

　　「畫坊？」

　　「上次你訓的 GAN——又都畫同一款花了。」

　　費倫皺眉。

　　「模式崩潰……我以為加大樣本就好。」

　　「加了也崩。」芙莉蓮放下茶杯，「——下一個，換個損失。」

---

## 芙莉蓮的冒險筆記

| 項目 | Edge Deployment | Phased Rollout | Batch Size & GPU Memory |
|:--|:--|:--|:--|
| 中文暱稱 | 邊緣部署 | 漸進式部署 | 批次規模與顯存調度 |
| 年代 | 2016 | — | — |
| 一句話口訣 | 模型搬到靠近資料的裝置 | 從單一專科起步，先穩再擴 | GPU 爆了減批次、加分片 |

**考試重點速記**
- 皆屬 **科目一｜部署策略**
- **Edge Deployment**：部署於靠近資料源（IoT／手機／邊緣裝置），降低延遲與頻寬，**減少雲端依賴**
- **Phased Rollout**：醫療場景應**從單一專科開始**，**逐步擴展至全院**
  - **iPAS 誤答**：從急診開始、全院同步
- **Batch Size & GPU Memory**：GPU 記憶體不足 → **減少 Batch Size + Data Sharding**
  - **iPAS 誤答**：減少資料量

**修塔爾克的戰士三對照**
| 戰士動作 | 工程對照 |
|:--|:--|
| 最常用裝備裝最外袋 | Edge Deployment（就近部署） |
| 先派一科新兵試線，穩了再擴 | Phased Rollout（漸進部署） |
| 背包太重就分成幾袋揹 | Batch Size ↓ + Data Sharding |

**心像崩潰對照（反面教材）**
- 錯誤：一夜鋪滿全院／急診先上 → 故障連鎖放大
- 錯誤：GPU 爆了砍資料量 → 少看病人，治錯病
- 正確口訣：「裝備就近、戰術漸進、背包減載分片。」
